# Markdown 转 PPT 大纲生成器

## 概述

本技能用于扫描当前项目中的所有 Markdown 文件，分析其内容和结构，然后生成一份结构化的中文 PPT 演示大纲。生成的大纲将提炼关键信息，按照逻辑顺序组织内容，适合直接用于制作 PowerPoint 演示文稿。

## 工作流程

### 第一步：扫描 Markdown 文件

使用 `search_file` 工具搜索项目中所有的 `.md` 文件：
- 设置 `pattern` 为 `*.md`
- 启用递归搜索（`recursive: true`）
- 从项目根目录开始搜索
- 工具会自动排除 `node_modules`、`.git` 等无关目录

示例：
```
search_file(
  target_directory: "/path/to/project",
  pattern: "*.md",
  recursive: true,
  caseSensitive: false
)
```

### 第二步：批量读取文件内容

使用 `read_file` 工具读取所有找到的 Markdown 文件：
- **重要**：使用并行工具调用，一次性读取多个文件以提高效率
- 记录每个文件的路径和完整内容
- 如果文件过大，可以分段读取

示例（并行读取）：
```
read_file(filePath: "/path/file1.md")
read_file(filePath: "/path/file2.md")
read_file(filePath: "/path/file3.md")
```

### 第三步：分析内容结构

对每个 Markdown 文件进行结构化分析，提取以下信息：

1. **标题层级**
   - 识别 H1（#）作为主要章节
   - 识别 H2（##）作为次级主题
   - 识别 H3-H6 作为详细要点

2. **关键内容元素**
   - 重要列表和要点
   - 代码块的用途说明（不包含具体代码）
   - 图片和图表的说明文字
   - 强调的文本（粗体、斜体）
   - 引用的重要观点

3. **逻辑关系**
   - 识别内容的因果关系
   - 发现步骤性的流程
   - 提炼核心概念和定义

### 第四步：生成 PPT 大纲

根据分析结果，生成结构化的中文 PPT 大纲，遵循以下格式：

```markdown
# [项目名称] - PPT 演示大纲

生成日期：[当前日期]
来源文件：[列出所有 Markdown 文件]

---

## 第一部分：封面页

**幻灯片 1：标题页**
- 主标题：[提取的项目或主题名称]
- 副标题：[简短描述]
- 演示人：[待填写]
- 日期：[当前日期]

---

## 第二部分：目录页

**幻灯片 2：内容概览**
- [主题 1]
- [主题 2]
- [主题 3]
- [主题 4]
- 总结与展望

---

## 第三部分：主体内容

**幻灯片 3：[第一个主题名称]**
- 要点 1：[简洁描述]
- 要点 2：[简洁描述]
- 要点 3：[简洁描述]
- 关键数据/图表：[如有]

**幻灯片 4：[第二个主题名称]**
- 要点 1：[简洁描述]
- 要点 2：[简洁描述]
- 要点 3：[简洁描述]

[继续添加更多幻灯片...]

---

## 第四部分：总结页

**幻灯片 N-1：核心要点总结**
- 关键发现 1
- 关键发现 2
- 关键发现 3
- 主要结论

---

## 第五部分：结束页

**幻灯片 N：致谢与问答**
- 感谢观看
- 问题与讨论
- 联系方式：[待填写]

---

## 备注说明

### 建议的视觉元素
- [列出适合添加的图表类型]
- [推荐的配色方案]
- [关键幻灯片的布局建议]

### 演讲要点提示
- [针对重点幻灯片的演讲建议]
- [需要特别强调的内容]
```

### 第五步：大纲生成原则

在生成 PPT 大纲时，严格遵循以下原则：

1. **层次清晰**
   - 按照逻辑顺序组织内容
   - 使用清晰的章节划分
   - 确保内容流程连贯

2. **简洁明了**
   - 每页幻灯片控制在 3-5 个要点
   - 使用短句和关键词
   - 避免冗长的段落

3. **突出重点**
   - 提炼核心概念
   - 去除过于技术性的细节
   - 保留最有价值的信息

4. **视觉友好**
   - 建议适合 PPT 呈现的格式
   - 标注可视化的数据
   - 提示需要图表的位置

5. **专业表达**
   - 使用准确的中文术语
   - 保持正式的语言风格
   - 统一表述方式

6. **适度容量**
   - 总幻灯片数量控制在 15-30 页
   - 避免信息过载
   - 预留讨论和问答时间

## 多文件整合策略

当项目包含多个 Markdown 文件时，采用以下策略整合内容：

### 1. 按主题分组
- 识别相关主题并合并同类内容
- 按照文件名或目录结构组织章节
- 建立主题之间的逻辑连接

### 2. 建立叙事流程
- 确定演示的故事线
- 从总体概述到具体细节
- 保持逻辑的连贯性

### 3. 避免重复
- 合并相似的内容
- 统一相同概念的表述
- 交叉引用而不是重复说明

### 4. 优先级排序
- 将最重要的内容放在前面
- 次要信息可以放在附录或备注
- 确保核心信息得到充分展示

## 输出格式

最终输出应该是一份完整的 Markdown 格式的 PPT 大纲文件，包含：

1. **完整的大纲结构** - 从封面到结束页的所有幻灯片规划
2. **详细的内容要点** - 每页幻灯片的具体内容
3. **备注和建议** - 演讲提示和视觉设计建议
4. **可执行性** - 可以直接基于此大纲制作 PPT

## 使用示例

**用户请求示例：**
- "帮我读取项目中的 Markdown 文件，生成一份 PPT 大纲"
- "我需要基于这些文档准备一个演示，请生成 PPT 大纲"
- "把项目文档转换成 PowerPoint 演示的大纲"
- "生成项目介绍的 PPT 大纲"

**执行时的关键步骤：**
1. 确认项目目录
2. 并行扫描和读取所有 `.md` 文件
3. 分析和提炼关键信息
4. 按照模板生成结构化大纲
5. 提供大纲文件供用户使用

## 注意事项

- 如果项目中没有找到 Markdown 文件，应告知用户并询问是否需要搜索其他格式的文档
- 对于非常大的 Markdown 文件（>10000 行），可以先读取部分内容，了解结构后再决定如何处理
- 生成的大纲应该是独立的文档，不依赖于原始 Markdown 文件
- 建议在大纲中标注哪些内容来自哪个源文件，方便后续追溯
